The AI Agent Node Overview

n8n has integrated LangChain directly into the visual canvas. This means we no longer have to write complex Python orchestration scripts to build autonomous agents. We can drag, drop, and wire together an LLM, long-term memory, and interactive tools in under five minutes.

The Anatomy of an n8n Agent

To build an AI Agent, we don't just use one node. We assemble a "stack" of specialized nodes attached to a central coordinator called the AI Agent node.

  • AI Agent Node: The brain. It natively understands how to prompt an LLM, parse its response, decide if a tool is needed, execute the tool, and recursively ask the LLM for the next step until the user's prompt is answered.
  • Model Nodes: We connect an LLM node (like OpenAI, Google Gemini, or Anthropic Claude) into the Model input of the AI Agent. We configure the API key and specific model version (e.g., gpt-4o) here.
  • Memory Nodes: We attach these to the Memory input so our agent remembers what was said three messages ago.
  • Tool Nodes: We attach these to the Tools input to give our LLM hands to interact with the real world (e.g., "Search the web" or "Query Postgres").

Connecting Models

Because n8n uses a standardized interface, swapping the actual brain of our agent takes two seconds. If OpenAI goes down, we literally delete the OpenAI node, drag a Google Gemini node onto the canvas, plug it into the same AI Agent node, and our workflow instantly recovers.

Structured Output: When building data pipelines, we can configure our Model nodes to force strict JSON output. This guarantees that when the LLM parses an invoice, it always returns a perfectly parsable { "total": 100 } object to the next n8n node, rather than a chatty paragraph.
⚠️ Warning: Every AI Agent execution costs real money. A recursive agent with 3 tools can easily make 8-12 LLM calls per single user message. Always set a Max Iterations limit (e.g., 5) on the AI Agent node to prevent runaway API bills.
💡 Tip: Always test AI agents using the Chat Trigger node first. It provides an instant chat UI inside the n8n editor — no need to wire up Slack or a frontend just to debug our prompts.

Giving the Agent Memory

LLMs are inherently stateless. If we say "My name is Alice", and then "What is my name?" in a new request, it won't know. To fix this, we wire a Chat Memory node into the AI Agent.

  • Window Buffer Memory: Keeps exactly the last X messages (e.g., 10). It is highly token-efficient but forgets ancient history.
  • Summary Memory: Instead of holding raw chat logs, it uses the LLM in the background to constantly distill the entire conversation into a running summary paragraph. It uses fewer tokens than a massive buffer while maintaining context spanning hundreds of turns.

Tool Nodes: Giving the Agent Hands

An LLM is useless if it can't execute actions. We can plug specialized nodes into the Agent's Tools input:

  • HTTP Request as Tool: We give the LLM an API endpoint (e.g., a Weather API) and tell it in the node description: "Use this to find the weather for a given city." The LLM will autonomously construct the API request, pause, wait for n8n to fetch the data, read the JSON response, and formulate an answer.
  • Code Node as Tool: We can allow the LLM to run sandboxed JavaScript.
  • Sub-Workflow as Tool: The ultimate abstraction. We build a massive 50-node n8n workflow that queries a database, scrapes a webpage, and formats a PDF. We then give that entire workflow to our AI Agent as a single tool named "Generate Financial Report".

Building a Basic Agent

To test agents locally, n8n provides a special trigger: the Chat Trigger node. It spins up a beautiful, floating chat UI directly inside the n8n editor. We type a message, the trigger fires, the AI Agent processes it, and the response streams right back into our editor chat UI.

Real Example: Document Processing Pipeline

We receive 50 PDF resumes a day and need to extract the applicant's name, email, and years of experience into Postgres.

  1. Start with a Webhook that receives the PDF file as a binary payload.
  2. Use the Extract From File node to convert the binary PDF into raw text.
  3. Pass the extracted text to a Basic LLM Chain node (a simplified agent).
  4. Configure the LLM node to enforce JSON Output and prompt it: Extract name, email, and years_experience from the following text: {{ $json.text }}.
  5. Connect a Postgres node. Map {{ $json.name }} and {{ $json.email }} directly into a SQL INSERT statement.

We just built an AI-powered OCR and parsing pipeline without writing a regex string.

Real Example: Agent with HTTP Tools

We want a Slackbot that can check our Stripe revenue when management talks to it.

  1. Start with a Slack node listening for mentions.
  2. Connect it to an AI Agent node.
  3. Attach an OpenAI Chat Model to the Agent's Model input.
  4. Attach a Window Buffer Memory node (backed by Redis to persist across crashes) to the Memory input.
  5. Attach an HTTP Request Tool node to the Tools input. Configure it to hit api.stripe.com/v1/balance. Give the tool a description: "Call this when asked about revenue, balance, or money."
  6. The Agent receives the Slack message. It decides to call the Stripe tool entirely on its own, parses the resulting Stripe JSON balance, converts it to plain English, and sends it back out to Slack.
