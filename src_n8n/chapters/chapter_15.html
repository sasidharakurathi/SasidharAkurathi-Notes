Chapter 15: The n8n CLI & REST API

The n8n CLI

Behind the beautiful Vue.js editor sits a powerful command-line interface. If we are running n8n globally via npm, we use the n8n command directly. If we are running the production Docker setup from Chapter 14, we run commands inside the running container.

# Executing a CLI command inside our running Docker container
docker exec -it n8n n8n [command]

The three most critical commands are start (which boots the server), export, and import.

Workflows as JSON (Version Control)

In a professional engineering environment, we don't rely solely on Postgres database backups to save our work. We treat workflows as code. An n8n workflow is ultimately just a giant, highly structured JSON array of node definitions and connections.

CLI Command Operation Details
n8n export:workflow --all Dumps every single workflow in the database into JSON files in the current directory.
n8n export:credentials --all Exports credentials. They are encrypted with the N8N_ENCRYPTION_KEY!
n8n import:workflow --input=file.json Reads a JSON file from disk and inserts it directly into the running n8n database.
💡 Tip: Because workflows are just JSON, we can track them with Git. When a developer makes a change to the "Stripe Sync" workflow in the editor, they should export it to JSON, commit it, and open a Pull Request. This gives us full git diff history of exactly which node parameters were changed.

The n8n Public REST API

n8n doesn't just call other APIs. It exposes its own Public REST API so our backend software can control the n8n application programmatically.

To enable it, we must first go to Settings → n8n API in the UI and generate an API key. This key belongs in our backend's .env file.

⚠️ Warning: By default, the Public REST API is disabled in self-hosted instances. We must ensure the environment variable N8N_PUBLIC_API_DISABLED=false is set in our docker-compose.yml before the API routes will resolve.

The API gives us full CRUD (Create, Read, Update, Delete) access over:

  • /api/v1/workflows: List all workflows, read their JSON structure, or create new ones dynamically.
  • /api/v1/executions: Query the history logs to see if a specific workflow crashed in the last 10 minutes.
  • /api/v1/credentials: Programmatically inject new API keys into n8n without opening the UI.

Triggering Workflows Programmatically

While we can always use a standard Webhook node to trigger workflows, n8n also provides a dedicated API endpoint to execute workflows by their internal ID.

When should we use the standard Webhook node versus the REST API execution endpoint?

  • Use standard Webhooks when: An external SaaS (like Shopify or GitHub) needs to blindly throw event data at n8n.
  • Use the REST API when: Our custom backend (e.g., FastAPI, Express) is deeply orchestrating n8n and needs to trigger a flow securely and specifically using the authenticated API Key.

Real Example: Shell Script to Git Commit Workflows

We do not want developers manually running docker exec commands every day. We build a simple bash script (backup.sh) to export all workflows, format the JSON beautifully, and push it to GitHub.

#!/bin/bash
# backup_workflows.sh

echo "Exporting n8n workflows from Docker container..."
# Create a dumps folder
mkdir -p ./n8n_backups

# Tell the n8n container to export all workflows into the mounted volume
docker exec n8n n8n export:workflow --backup --output=/home/node/.n8n/git_backups/

# Move the files from the docker volume to our local git repo
cp -r /var/lib/docker/volumes/n8n_data/_data/git_backups/* ./n8n_backups/

echo "Committing to Git..."
git add ./n8n_backups/
git commit -m "chore(n8n): Daily automated workflow definition backup"
git push origin main
echo "Done!"

Real Example: Trigger n8n from FastAPI

Our FastAPI backend handles user registration and creates the Postgres database record. However, we want n8n to handle all the "fluff" — subscribing them to Mailchimp, pinging Slack, and generating a welcome PDF. We use the n8n REST API to trigger the workflow directly from Python.

# app/services/n8n_service.py
import httpx
import os

N8N_URL = os.getenv("N8N_URL") # e.g., "https://n8n.mycompany.com"
N8N_API_KEY = os.getenv("N8N_API_KEY")

async def trigger_onboarding_workflow(user_id: int, email: str):
    """
    Triggers the n8n workflow (ID: 42) responsible for Mailchimp and Slack.
    Uses the official n8n REST API, NOT a standard webhook.
    """
    url = f"{N8N_URL}/api/v1/workflows/42/execute"
    
    headers = {
        "X-N8n-Api-Key": N8N_API_KEY,
        "Accept": "application/json"
    }
    
    # This data will be available in n8n inside the exact trigger node
    payload = {
        "data": {
            "user_id": user_id,
            "email": email,
            "source": "fastapi_backend"
        }
    }
    
    async with httpx.AsyncClient() as client:
        response = await client.post(url, json=payload, headers=headers)
        response.raise_for_status()
        
        return response.json()
