Chapter 12: Most Useful Integrations for Developers

The Big 5 Core Nodes

Some integrations are so fundamental we use them in almost every workflow. Master these five before looking at anything else:

  • Webhook: Our ingress point. It creates an active URL that instantly accepts incoming HTTP GET, POST, or PUT requests from internal systems or SaaS platforms and kicks off a workflow.
  • HTTP Request: Our egress point. When n8n doesn't have an official node for an app, we construct the raw API call here. It handles pagination, OAuth2, and binary file streaming effortlessly.
  • PostgreSQL / MySQL / MongoDB: Our state management. By connecting our production or staging databases directly to n8n, we can run complex raw queries (e.g., UPDATE users SET active = false WHERE id = $1), injecting parameterized data dynamically without writing CRUD endpoints.
  • Redis: Essential for tracking state across workflows, managing incredibly fast idempotency checks, or buffering high-volume data before writing it to a slow relational database.
⚠️ Warning: When connecting Postgres/MySQL nodes directly to production databases, always use a read-only credential for query-only workflows. A single typo in a manual SQL expression (DELETE FROM users instead of DELETE FROM users WHERE id = 42) can wipe our entire table.
💡 Tip: If an official n8n integration node doesn't support an action we need (e.g., Stripe's "Search Subscriptions" endpoint), don't wait for n8n to add it. Use the HTTP Request node with our Stripe API key to call the endpoint directly. The HTTP node is always our fallback.

The Cloud Office Stack

Automating interactions with non-technical departments usually involves leaning on "Office" SaaS.

  • Google Sheets: Functions beautifully as a poor man's database. We can append rows instantly via form submissions, read massive sheets to kick off batch processes, or update specific rows by matching a primary key dynamically.
  • Gmail / Outlook: Not just for sending SMTP alerts. We can configure triggers to poll our support inbox, download email attachments automatically, run them through an OCR node, and insert them into AWS S3.
Credential Isolation: When connecting to Google Workspace or Office 365, do not use a personal employee service account. Create a dedicated "automationbot@ourcompany.com" address. If an employee leaves, their disabled accounts will instantly crash our critical n8n workflows.

Communication Layers

Alerts need to be visible, fast, and interactive.

  • Slack: We can post rich-text block messages, funnel errors into #engineering-alerts, or even listen for specific Slack commands to trigger n8n workflows backwards from the chat window.
  • Telegram Bot: One of the easiest ways to build a complete mobile interface for our workflows. We can construct a BotFather account in 3 minutes, have n8n listen for Telegram commands, and respond to users natively on their phones without building an iOS app.

Project Management & DevTools

We often need business logic to sync across engineering and operations.

  • GitHub: We can sync issues, automate PR labeling, or trigger a massive CI/CD testing pipeline in n8n whenever a push event happens on the main branch.
  • Airtable: A beloved tool for operations teams. We can use n8n to listen for new rows in Airtable and instantly push that data to our real Postgres database, keeping non-technical ops teams and engineering databases perfectly synchronized.
  • Notion: We can automate the creation of rich architectural docs, update sprint boards when GitHub PRs merge, or dump raw customer feedback logs into searchable databases.

The AI Era (OpenAI)

n8n has fully embraced LLMs. The OpenAI nodes (and Anthropic, etc.) allow us to inject intelligence into our pipes without writing complex OpenAI Python SDK wrappers.

We can drag an OpenAI node onto the canvas, feed it a block of messy customer feedback text from Zendesk, give it a system prompt ("Extract the sentiment, bug type, and urgency"), and n8n will pass us back clean, structured JSON ready to insert into Jira or Postgres.

Real Example: Issue Triaging Pipeline

Our product team uses Notion, our developers use GitHub, and everyone watches Slack. Let's synchronize them.

  1. Trigger: We use the GitHub Trigger node to listen for issue_opened events on our target repository.
  2. We drop an OpenAI node. We pass the raw GitHub issue text to it and prompt it to categorize the issue as "Frontend", "Backend", or "DevOps", outputting strict JSON.
  3. We add a Notion node. It takes the parsed JSON and creates a beautiful new page in the Product team's "Triage Board" database, tagging it with the correct category.
  4. We add a Slack node. We construct a message: 🚨 New {{ $json.category }} issue: {{ $json.title }} and pipe it into #dev-triage.

A complete cross-platform sync loop built in under 15 minutes.

Real Example: Form to Sheets to Email

We are launching a quick beta signup form and don't want to build a whole Postgres user registration API yet.

  1. Trigger: A Webhook node listening for POST requests from our Next.js frontend form.
  2. We use the Google Sheets node set to Append Row. We map the webhook payload's {{ $json.body.email }} and {{ $json.body.name }} into our "Beta List" spreadsheet columns.
  3. We use the Gmail node to send an immediate welcome email back to the user. We map the recipient address directly to {{ $json.body.email }} from the initial webhook.
  4. We run a Slack node immediately alongside the Gmail node to ping #sales that a new beta user just signed up.

We just built a fully functional marketing backend with exactly zero lines of code, and we can refactor the Google Sheets node to a Postgres node in 30 seconds when it's time to scale.
