Important Expressions & Variables

n8n expressions use standard JavaScript inside double curly braces {{ }}. Here is a cheat sheet of the built-in variables we reach for daily.

Variable / Expression What It Does
{{ $json.propertyName }} Accesses the current item's JSON property.
{{ $binary.data.fileName }} Accesses metadata of an attached binary file (e.g., an uploaded PDF).
{{ $parameter.url }} Reads the value we manually typed into the node's 'url' field.
{{ $now }} Luxon DateTime object for right now. E.g. {{ $now.toISODate() }}
{{ $execution.id }} The unique string ID of the current running workflow execution.
{{ $('NodeName').item.json.id }} Reaches back in time to get the matching item's JSON from a previous node.
💡 Tip: If an expression looks too messy visually on the canvas (e.g., a heavily nested ternary operator {{ $json.age > 18 ? 'Adult' : ($json.age > 12 ? 'Teen' : 'Child') }}), push that logic back into a proper Javascript `Code` node.

Trigger Types Recap

Trigger Type How It Fires Best For
Webhook External HTTP request hits n8n URL Real-time events from GitHub, Stripe, Slack
Schedule (Cron) Internal timer fires at set intervals Batch jobs, daily reports, periodic sync
App Trigger (Instant) n8n auto-registers webhook with the app Slack messages, GitHub events, Stripe payments
App Trigger (Polling) n8n checks the app's API periodically Legacy APIs without webhook support

Common Node Patterns

We see these same structural patterns in almost every production workflow:

  • IF vs Switch: Use IF when checking a single truthy/falsy condition (e.g., "Is user active?"). Use Switch when checking ENUMs (e.g., Status is "Draft", "Published", or "Archived").
  • Merge (By Key): The "V-shape" pattern. Two API calls happen in parallel (e.g., Get Users from Shopify, Get Users from Postgres), followed by a Merge node to join them on their email address.
  • Looping (Split in Batches): Processing 500 records by sending them to a "Split" node configured to size 1, running an HTTP request, then routing the output back into the Split node until it finishes.

Common Errors & Fixes

When our workflows break, it's usually because of these five culprits:

Error Message Root Cause / Fix
"Workflow could not be activated" The App Trigger node failed to register the webhook with the 3rd party (e.g., GitHub). Check if our API credentials have exact webhook creation permissions.
"Webhook not registered" (404) We hit the Production webhook URL (/webhook/) but the workflow toggle switch in the UI is set to Inactive.
"Execution timed out" A node (usually a massive DB query or hanging HTTP request) exceeded the global EXECUTIONS_TIMEOUT_MAX=XYZ setting.
"Credential not found" We imported a raw workflow JSON file into a new server, but didn't import the matching credential objects (which are exported separately for security).
"Cannot read property of undefined" We used {{ $json.user.name }} in an expression, but the user object doesn't exist in this specific item's payload. Always use optional chaining in the Code node (item.user?.name) if payloads are volatile.

CLI & Docker Commands Cheat Sheet

Keep these commands handy when managing the n8n infrastructure via SSH.

# === n8n CLI Commands (run inside container) ===

# Export EVERYTHING (workflows and credentials) to a mounted folder
docker exec -it n8n n8n export:workflow --all --output=/home/node/.n8n/backup/
docker exec -it n8n n8n export:credentials --all --output=/home/node/.n8n/backup/

# Import exactly one workflow from a JSON file
docker exec -it n8n n8n import:workflow --input=/home/node/.n8n/backup/workflow-1.json

# Execute a specific workflow strictly from the command line 
docker exec -it n8n n8n execute --id=42
# === Docker Management Commands ===

# View the raw console logs (tail the last 100 lines) of the main process
docker logs n8n -f --tail 100

# View queue/worker logs specifically
docker logs n8n-worker -f 

# Gracefully restart the stack without dropping queued jobs immediately
docker compose restart

# Scale generic workers on the fly (Redis queue mode required)
docker compose up -d --scale n8n-worker=4
⚠️ Warning: Running docker compose down immediately kills in-flight executions. Always issue a docker compose stop and wait for containers to gracefully exit before tearing down the stack.
