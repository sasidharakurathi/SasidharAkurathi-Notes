Chapter 7: Working with the Code Node

When to Reach for the Code Node

n8n's visual nodes (like IF, Switch, Set, and Item Lists) cover 90% of automation needs. We should reach for the Code node when:

  • We need to perform complex data deduplication or aggregation (e.g., reducing an array of invoices by customer_id).
  • We have to heavily reshape deeply nested, proprietary JSON API responses into flat structures.
  • We need to generate cryptographic hashes (like HMAC-SHA256) for a custom webhook signature.

If we find ourselves writing a 10-line ternary operator inside an expression {{ }}, delete it and write clean JavaScript in a Code node instead.

JavaScript vs Python Mode

By default, the Code node runs sandboxed JavaScript (Node.js). If we flip the "Mode" toggle to "Python", n8n executes the code using Pyodide (Python compiled to WebAssembly).

Developer Warning: Python mode is highly experimental and executed in the browser/WASM context. We cannot import pandas or requests. Stick to JavaScript unless we are forced to translate a hyper-specific Python standard library snippet.

Input Data: $input.all() vs $input.first()

Everything hitting our Code node is an array of Items. We access them using the $input global object.

  • $input.all() returns the entire array of items. Useful for aggregations, sorting, or looping over every row.
  • $input.first() returns only the index 0 item. Useful if our previous node was guaranteed to only return an array of 1 item (like a Webhook trigger) to avoid writing items[0].json everywhere. 
// Getting ALL items (Standard pattern)
const items = $input.all();

// items is an array of objects structured exactly like:
// [ { json: { name: "Alice" } }, { json: { name: "Bob" } } ]

Returning Data: The Cardinal Rule

The Code node MUST always return an array of objects, and those objects MUST have a json key.

If we violate this structural rule, n8n will crash the node. We cannot just return a raw string or a raw array of strings.

// ❌ BAD: Returning a raw string
return "Success";

// ❌ BAD: Returning an array of raw objects
return [{ user: "Alice" }];

// ✅ GOOD: Returning a proper n8n Item format
return [{ json: { status: "Success", user: "Alice" } }];

If we are simply iterating over and mutating the existing items (e.g., adding a field), returning the mutated original items array handles this structure for us automatically:

const items = $input.all();
for (const item of items) {
  item.json.processedAt = new Date();
}
return items; // Already correctly structured!

Accessing Environment Variables

Unlike the expression editor's $env("KEY"), the Code node (in JavaScript mode) allows us to securely access our VPS environment configurations using a direct object:

const stripeKey = $env.STRIPE_SECRET_KEY;
const isProd = $env.NODE_ENV === 'production';

Error Handling inside the Code Node

Because the Code node runs standard JS, we should wrap risky logic (like native JSON.parse() on webhook data) in try/catch blocks. However, if we want the workflow to fail entirely, deliberately throw an Error.

const items = $input.all();

if (items.length === 0) {
  // Force a crash that will bubble up to n8n's visual Error logger
  throw new Error("Critical Failure: No items received from previous database query");
}

return items;

Using External Libraries (NPM)

By default, external libraries are disabled for security. n8n natively exposes native Node crypto, url, and its own bundled luxon for time.

To use an external NPM library like lodash, we must do two things on our self-hosted instance:

  1. Add it to the environment file: NODE_FUNCTION_ALLOW_EXTERNAL=lodash,moment
  2. (If using Docker) Build a custom Dockerfile that runs npm install lodash on top of the base image.

Once allowed, we import them securely:

const _ = require('lodash');
const groups = _.groupBy($input.all(), 'json.departmentId');

Real Example: Parse & Reshape Complex JSON

Imagine a brutal legacy API returns: { "data": "Alice|Bob|Charlie", "codes": [1,2,3] }. We need an array of clean objects so the next node can insert them into Postgres.

const item = $input.first().json;
const names = item.data.split('|');
const output = [];

for (let i = 0; i < names.length; i++) {
  // Construct the new n8n Item format
  output.push({
    json: {
      fullName: names[i],
      accessCode: item.codes[i]
    }
  });
}

// Next node fires 3 times (once for Alice, Bob, and Charlie)
return output;

Real Example: Deduplicating Items

We fetched 10,000 logs and need to filter out duplicate customer IDs so we don't spam a webhook downstream.

const items = $input.all();
const seenIds = new Set();
const uniqueItems = [];

for (const item of items) {
  const id = item.json.customerId;
  
  if (!seenIds.has(id)) {
    seenIds.add(id);
    uniqueItems.push(item);
  }
}

return uniqueItems;

Debugging Tip: console.log()

We can use console.log() inside the Code node to debug loops. However, it does not display on the n8n visual canvas. It dumps directly to the Node.js standard output stream.

To view our logs, we must SSH into our server and run docker logs -f my_n8n_container while triggering the workflow.
