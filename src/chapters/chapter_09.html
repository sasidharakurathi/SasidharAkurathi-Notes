CHAPTER 9 - Extra Data Types

Beyond Strings and Integers

Until now, we have mostly used basic Python types like str, int, float, and bool. But real-world applications require far more complex data. We need to handle dates, exact monetary values, and uniquely generated IDs.

Normally, sending these over the web is incredibly annoying. JSON (the language of the web) only understands strings, numbers, booleans, and lists. It literally does not know what a "date" is. It just sees text.

FastAPI handles this translation for us automatically. We write our code using standard, powerful Python data types, and FastAPI silently converts them to and from simple JSON strings when talking to the browser.

Working with UUIDs

A UUID (Universally Unique Identifier) is a long, randomly generated string of letters and numbers (like 123e4567-e89b-12d3-a456-426614174000). In modern applications, we almost never use simple integers like 1, 2, or 3 for database IDs, because hackers could easily guess the next ID. We use UUIDs instead. 

from uuid import UUID
from fastapi import FastAPI

app = FastAPI()

@app.put("/items/{item_id}")
def update_item(item_id: UUID):
    return {"item_id": item_id}

Line 1: We import the standard UUID class that comes built-in with Python.

Line 7: We tell FastAPI the path parameter must be a valid UUID. If a user visits /items/apple, FastAPI will automatically reject it, realizing "apple" is not a valid 36-character UUID pattern.

In plain English: We use UUIDs to give database records unbreakable, un-guessable IDs. FastAPI automatically checks that the user's string is a valid UUID format before letting it pass.

Dates and Times

We often need to record exactly when a user signed up, or when an item expires. Python provides four brilliant tools for this inside the built-in datetime module: datetime, date, time, and timedelta.

  • datetime: A specific date and a specific time. (e.g., 2026-07-21T14:30:00)
  • date: Just the calendar day. (e.g., 2026-07-21)
  • time: Just the time on a clock, regardless of the day. (e.g., 14:30:00)
  • timedelta: A duration, or span of time. (e.g., 14 days, 3 hours)

Decimals and Bytes

When dealing with money or financial calculations, we should never, ever use float. Floats round numbers weirdly in computer memory (like saying $10.00 is actually $9.9999999999). Instead, we use the standard Python Decimal type, which is mathematically perfect.

We might also need to read raw, binary computer data (like reading a raw file before saving it). For this, we use the standard Python bytes type.

A Complete Example

Let's look at how beautifully FastAPI processes all of these complex Python types at the exact same time, automatically converting them to JSON for us.

from datetime import datetime, time, timedelta
from decimal import Decimal
from uuid import UUID

from fastapi import Body, FastAPI

app = FastAPI()

@app.put("/items/{item_id}")
def update_item(
    item_id: UUID,
    start_datetime: datetime = Body(),
    end_time: time = Body(),
    process_after: timedelta = Body(),
    price: Decimal = Body(),
):
    end_datetime = start_datetime + process_after
    return {
        "item_id": item_id,
        "start_datetime": start_datetime,
        "end_time": end_time,
        "process_after": process_after,
        "price": price,
        "expected_finish": end_datetime
    }

Let's break down the magic happening here:

  • Line 11: The item_id is a perfect UUID, pulled directly from the URL.
  • Line 12: We expect a full date and time string in the JSON body. FastAPI takes that string, and beautifully converts it into a real Python datetime object for us.
  • Line 13: We pull out just a time (like 14:30:00).
  • Line 14: We expect a duration. We might send a float across the internet (like 10.5 seconds), but FastAPI transforms it into a timedelta object before giving it to our code.
  • Line 15: We accept a price guarantee. We can send a string like "19.99" in the JSON, and FastAPI will convert it safely into a flawless Decimal object for our financial math.
  • Line 17: Because FastAPI converted these into real Python objects, we can actually do math with them! We simply add start_datetime + process_after to figure out exactly when our process will finish.
⚠️ Common Mistake: Trying to use float for money. Never do it! Always import Decimal from the decimal module to prevent missing pennies.
