The FastAPI() Constructor

When we instantiate our main app = FastAPI(), we can configure our entire application globally. Here are the most critical parameters we should know as senior developers:

  • title: The gigantic bold title permanently displayed at the top of our /docs page. (Default: "FastAPI").
  • description: A massive, Markdown-supported string explaining our API underneath the title.
  • version: The current API version (e.g., "1.0.0"). Very useful if we have a public-facing API.
  • docs_url: The path to our Swagger UI. Defaults to "/docs". Pro-tip: Set to None in production to completely disable and hide it from hackers!
  • redoc_url: The path to Redoc (an alternative API documentation UI). Defaults to "/redoc". Set to None to disable.
  • openapi_url: The path that serves the raw JSON schema of our entire API. Both Swagger and Redoc read this file to build their UIs. Defaults to "/openapi.json".
  • dependencies: A list of Depends() functions that will be aggressively executed globally before every single route in our entire application.
  • servers: A list of dicts defining exactly where this API is hosted (e.g., {"url": "https://api.ourstartup.com"}) for our OpenAPI clients to know.
  • lifespan: A single async context manager function we wrote to perfectly handle our app's startup and shutdown sequence (loading AI models, closing DBs).

The APIRouter() Constructor

When writing our users.py mini-apps using APIRouter(), we configure the entire "Department" at once:

  • prefix: A string (like "/users") that perfectly prefixes every single route inside this router. Note: It cannot end with a trailing slash!
  • tags: A list of strings (like ["Users", "Admin"]). This magically groups all the routes together under bold headings in the Swagger UI.
  • dependencies: A list of Depends() functions that are aggressively enforced on every single route uniquely inside this specific router.
  • responses: A dictionary of HTTP status codes and descriptions (like {404: {"description": "Not Found"}}) applied instantly to all routes in this router.

Path Operation Decorator Parameters

The beating heart of our app is the @app.get() or @router.post() decorators. Here is absolutely everything we can inject into them to master FastAPI.

# A masterclass in route configuration!
@app.post(
    path="/items/",
    response_model=schemas.Item,
    status_code=status.HTTP_201_CREATED,
    tags=["Inventory"],
    summary="Create a new shiny item",
    description="This route inserts a brand new item into the PostgreSQL database. **Markdown is completely supported!**",
    response_description="The successfully created item perfectly serialized.",
    deprecated=False,
    include_in_schema=True
)
async def create_item(item: schemas.ItemCreate):
    return item

The Full Breakdown:

  • path: (Required) the string route (e.g., "/users/{id}").
  • response_model: The Pydantic schema FastAPI must forcefully strictly validate against and serialize our return data into.
  • status_code: The exact HTTP numerical code to return if successful (use status.HTTP_200_OK, never hardcode numbers).
  • tags: Groups the route in Swagger UI.
  • dependencies: Specific dependencies just for this single route.
  • summary: A short, beautiful one-line title for the route inside Swagger UI.
  • description: An incredibly long, Markdown-rich explanation of exactly what this route does and why it exists.
  • response_description: A description of the exact return payload.
  • deprecated: If True, FastAPI magically strikes a line through the route in Swagger UI telling all our frontend devs to stop using it!
  • include_in_schema: If False, the route turns completely invisible in our public Swagger UI documentation (great for secret admin endpoints!).
  • response_model_exclude_unset: A brilliant boolean that strips away None or unset values from our response JSON to keep it tiny.
  • response_model_include / _exclude: Extremely specific sets telling FastAPI to literally cut out specific Pydantic fields (like hiding a password field).
