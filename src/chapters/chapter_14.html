CHAPTER 14 - File Uploads

Receiving Files with File()

Accepting a file upload (like a profile picture or a spreadsheet) sounds complicated, but FastAPI makes it incredibly simple. Because files are essentially just complex forms, we need the python-multipart package installed, just like we did in the last chapter.

The most basic way to receive a file is to use the File() function to pull the raw bytes directly into memory.

from fastapi import FastAPI, File

app = FastAPI()

@app.post("/files/")
def create_file(file: bytes = File()):
    return {"file_size": len(file)}

Line 6: We tell FastAPI that we expect a variable named file, that its type is raw bytes, and that it must be pulled from an uploaded File().

⚠️ Warning: Using bytes means FastAPI will read the entire file into the computer's RAM (memory). If ten users try to upload 1GB videos at the exact same time, our server will completely run out of memory and crash. Never use this for large files!

UploadFile - The Better Way

Because loading entire files into memory is dangerous, FastAPI gives us a much safer, much smarter tool: UploadFile. 

from fastapi import FastAPI, File, UploadFile

app = FastAPI()

@app.post("/uploadfile/")
def create_upload_file(file: UploadFile):
    return {"filename": file.filename}

Line 6: By changing the type hint to UploadFile, we unlock FastAPI's file management super powers.

UploadFile uses a "spooled" file system. If the file is small (under 1 MB), it safely holds it in memory. If a user uploads a massive 4GB video, FastAPI intelligently streams the data onto the server's hard drive piece by piece, preventing our server's memory from ever crashing.

In plain English: Always default to using UploadFile. It protects our server from crashing when users upload gigantic files, and gives us excellent built-in tools to manipulate the file.

UploadFile Attributes

When we accept an UploadFile, it comes with several incredibly useful properties attached to it that we can use immediately:

  • file.filename: The actual name of the file the user uploaded (e.g., "my_selfie.png").
  • file.content_type: The exact type of the file, like "image/jpeg" or "application/pdf".
  • file.size: The exact size of the file in bytes (if available).
  • file.file: The actual Python file-like object. We can pass this directly to libraries that expect a standard Python file.

Reading File Content Asynchronously

Because an UploadFile might be massive, reading its contents happens asynchronously. This means our server can go process other users' requests while waiting for the hard drive to finish reading the giant file.

@app.post("/uploadfile/")
async def create_upload_file(file: UploadFile):
    contents = await file.read()
    return {
        "filename": file.filename, 
        "file_size": len(contents)
    }

Line 2: Notice that we changed our function to an async def.

Line 3: We use the await file.read() command to safely read the contents. We can also use await file.seek(0) to go back to the start of the file, or await file.write(data) if we needed to inject more data into it.

Multiple File Uploads

What if a user wants to upload an entire photo album at once? We just use our trusty Python List type!

from typing import List
from fastapi import FastAPI, UploadFile

app = FastAPI()

@app.post("/uploadfiles/")
async def create_upload_files(files: List[UploadFile]):
    file_names = [file.filename for file in files]
    return {"filenames": file_names}

Line 7: By asking for a List[UploadFile], the Swagger documentation will automatically generate a button letting the user select multiple files at once.

Files + Form Data Together

In the last chapter, we learned a strict rule: we cannot mix HTML forms and JSON bodies. However, we can mix HTML forms and Files!

This is what happens when we fill out a profile page: we type in our text fields (Form) and upload an avatar image (File) at the exact same time.

from fastapi import FastAPI, File, Form, UploadFile

app = FastAPI()

@app.post("/profile/")
async def update_profile(
    username: str = Form(),
    avatar: UploadFile = File(...)
):
    return {
        "username": username,
        "avatar_name": avatar.filename
    }

Lines 7-8: We declare our username as a standard Form(), and our avatar as a File(...) (the ... just means it is strictly required).

FastAPI will automatically declare this route as a "multipart" form, perfectly splitting the text fields from the massive file chunks.

File Limits and Validation Tips

Right now, our code allows users to upload any file they want. What if an evil bot tries to upload a 50GB file named virus.exe when we only wanted a PDF?

Currently, FastAPI does not have a native, built-in parameter to limit file sizes or force a specific file extension directly in the route definition. We have to validate this manually inside our function.

from fastapi import HTTPException

@app.post("/upload-pdf/")
async def upload_pdf(file: UploadFile):
    # 1. Validate the file type
    if file.content_type != "application/pdf":
        raise HTTPException(
            status_code=400, 
            detail="Only PDF files are allowed"
        )
        
    # 2. Validate the size (rough estimation by chunking)
    file_size = 0
    while chunk := await file.read(1024 * 1024): # Read 1MB at a time
        file_size += len(chunk)
        if file_size > (5 * 1024 * 1024): # 5 Megabytes
            raise HTTPException(
                status_code=400, 
                detail="File too large. Max size is 5MB"
            )
            
    return {"filename": file.filename, "size": file_size}

By manually checking the content_type and carefully reading the file in chunks to measure the size, we can protect our server from dangerous payloads!
