1. Decorators & Routing

from fastapi import FastAPI, APIRouter

app = FastAPI()
router = APIRouter(prefix="/users", tags=["Users"])

@app.get("/items/{id}")        # Read data
@app.post("/items/")           # Create data
@app.put("/items/{id}")        # Replace entire item
@app.patch("/items/{id}")      # Update partial item
@app.delete("/items/{id}")     # Delete item
@app.options("/items/")        # Check allowed methods
@app.head("/items/")           # Check headers (no body)

# Include a router into the main app
app.include_router(router)

2. All Parameter Types

Where does the data come from? FastAPI imports specific classes to map precisely where to look.

from fastapi import Path, Query, Body, Header, Cookie, Form, File, UploadFile

@app.post("/everything/{item_id}")
async def massive_route(
    # 1. URL Path: /everything/42
    item_id: int = Path(..., title="The ID", ge=1),

    # 2. URL Query: ?q=search&limit=10
    q: str = Query(None, min_length=3),
    limit: int = Query(10, le=100),

    # 3. JSON Body: {"name": "Apple"}
    item: ItemSchema = Body(...),
    
    # 4. HTTP Headers: X-Token: "abc"
    x_token: str = Header(None),
    
    # 5. Browser Cookies: session_id=123
    session_id: str = Cookie(None),
    
    # 6. HTML Form Data: text input fields
    username: str = Form(...),
    
    # 7. File Uploads: multipart/form-data
    profile_pic: UploadFile = File(...)
):
    return {"message": "Success"}

3. Common Response Patterns

How do we send data back? Usually just return dict, but we often need custom responses.

from fastapi import Response, status
from fastapi.responses import JSONResponse, HTMLResponse, RedirectResponse

# 1. Standard dict (FastAPI converts to JSON instantly)
@app.get("/basic")
def get_basic():
    return {"hello": "world"}

# 2. Custom Status Code (201 Created)
@app.post("/item", status_code=status.HTTP_201_CREATED)
def create():
    return {"status": "created"}

# 3. Direct JSONResponse (To set a custom cookie or header instantly)
@app.get("/login")
def login_cookie():
    response = JSONResponse(content={"message": "Logged in"})
    response.set_cookie(key="session", value="fake-token")
    return response

# 4. Redirecting the User
@app.get("/old-link")
def redirect_me():
    return RedirectResponse(url="/new-link", status_code=status.HTTP_301_MOVED_PERMANENTLY)

4. Dependency Injection Quick Reference

Dependencies let us reuse code gracefully.

from fastapi import Depends

# 1. A basic dependency function
def get_db():
    db = SessionLocal()
    try:
        yield db
    finally:
        db.close()

# 2. Injecting it into a route
@app.get("/users/")
def get_users(db: Session = Depends(get_db)):
    return db.query(User).all()

# 3. Router-Level Dependency (Enforced on ALL routes in the router)
router = APIRouter(dependencies=[Depends(verify_token)])

5. Auth Quick Reference

The standard pattern for protecting an endpoint with OAuth2 Password Bearer.

from fastapi import Depends, HTTPException, status
from fastapi.security import OAuth2PasswordBearer
from pydantic import BaseModel

# 1. Tells Swagger UI where to send the login data
oauth2_scheme = OAuth2PasswordBearer(tokenUrl="token")

# 2. Our standard dependency to verify the token
async def get_current_user(token: str = Depends(oauth2_scheme)):
    if not token == "fake-secret-token":
        raise HTTPException(
            status_code=status.HTTP_401_UNAUTHORIZED,
            detail="Invalid authentication credentials",
            headers={"WWW-Authenticate": "Bearer"},
        )
    return {"username": "alice"}

# 3. Protecting an endpoint
@app.get("/users/me")
async def read_users_me(current_user: dict = Depends(get_current_user)):
    return current_user

6. Pydantic Schema vs SQLAlchemy Model

⚠️ Absolute Critical Reminder:
SQLAlchemy Models represent the permanent structure of our physical PostgreSQL tables. They inherit from Base.

Pydantic Schemas represent the temporary shape of the JSON travelling over the internet. They inherit from BaseModel.

We must use Schemas in our FastAPI routes (response_model=ItemSchema), and write Python code to literally convert the Database Model into the Schema. Never return a raw SQLAlchemy Model over the internet!
