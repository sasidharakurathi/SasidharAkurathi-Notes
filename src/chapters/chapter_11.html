What is a Header?

When our browser sends a request to a server, it does not just send the URL and the body. It also sends a hidden list of metadata called "Headers".

Think of headers like the stamp on an envelope. We don't read the stamp to understand the letter, but the post office uses the stamp to figure out how to handle the delivery. Headers tell the server things like what type of browser we are using, what language we prefer, or a secret token proving we are logged in.

In plain English: Headers are hidden key-value pairs of metadata sent along with every web request. They contain technical context about the request.

Reading Headers with Header()

Reading a header in FastAPI is just as simple as reading a query or a cookie. We import Header from the main fastapi package.

from fastapi import FastAPI, Header

app = FastAPI()

@app.get("/items/")
def read_items(user_agent: str = Header(default=None)):
    return {"User-Agent": user_agent}

Line 6: By using = Header(default=None), we instruct FastAPI to specifically look inside the hidden HTTP headers for our variable.

The Underscore-to-Hyphen Magic

Did we notice something strange in the example above? Our Python variable is named user_agent (with an underscore), but the standard HTTP header we are trying to read is actually named User-Agent (with a hyphen).

FastAPI performs a magical, automatic conversion here. Because hyphens (-) are completely illegal in Python variable names, we cannot name our variable User-Agent. FastAPI solves this by automatically converting any underscores in our variable names into hyphens when it searches the HTTP headers!

Not only that, HTTP headers are supposed to be case-insensitive. So USER_AGENT, user_agent, and User_Agent will all successfully catch the User-Agent header. It just works.

⚠️ Common Mistake: Trying to disable the magic. If for some weird reason we actually need to read a custom header that literally has an underscore in its name (like strange_header), we must explicitly tell FastAPI to stop converting it by using Header(convert_underscores=False).

Duplicate or Multiple Headers

Sometimes, a browser will send the exact same header multiple times in one request, but with different values. For example, a user might send multiple X-Token headers.

If we want to capture all of them, we just declare our variable as a Python List.

from typing import List
from fastapi import FastAPI, Header

app = FastAPI()

@app.get("/items/")
def read_items(x_token: List[str] = Header(default=None)):
    return {"X-Token values": x_token}

Line 7: By using List[str], FastAPI will find every single X-Token header in the request and gather them seamlessly into a clean list for us.

Header Models with Pydantic

Just like we did with cookies and request bodies, if we expect a massive list of custom headers, we can group them all into an ultra-strict Pydantic model.

from pydantic import BaseModel
from fastapi import FastAPI, Header

class CommonHeaders(BaseModel):
    host: str
    save_data: bool
    x_custom_token: str

app = FastAPI()

@app.get("/items/")
def read_items(headers: CommonHeaders = Header()):
    return headers

Line 12: We use = Header() to instruct FastAPI to build the entire CommonHeaders model exclusively out of incoming HTTP headers.

Common Headers We Will See

While we can invent our own headers (usually starting with X-, like X-Secret-Token), here are the most common standard headers we might want to read:

  • User-Agent: Tells us what browser and operating system the user has.
  • Authorization: Contains the security tokens used for logging in.
  • Accept-Language: Tells us whether the user prefers to read the site in English, Spanish, etc.
  • Host: Tells us the exact domain name the user requested.
