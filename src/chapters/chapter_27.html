Returning JSONResponse Directly

Normally, we just return a Python dict or a Pydantic model, and FastAPI automatically converts it into JSON behind the scenes. But sometimes, we need absolute control over the exact response being built.

If we want to bypass FastAPI's automatic conversion (maybe we are building a completely weird, custom error handler), we can return a JSONResponse directly.

from fastapi import FastAPI
from fastapi.responses import JSONResponse

app = FastAPI()

@app.get("/custom-json")
def get_custom_json():
    # We are returning the raw Response object, not a dict!
    return JSONResponse(
        content={"message": "I built this response manually!"},
        status_code=201,
        headers={"X-Custom-Header": "Hello"}
    )

Notice how building the response manually allows us to inject custom HTTP headers and status codes in exactly one step.

HTMLResponse & PlainTextResponse

APIs do not have to return JSON! If we want to return a raw HTML string so the browser renders a webpage, we use HTMLResponse. 

from fastapi.responses import HTMLResponse, PlainTextResponse

@app.get("/html", response_class=HTMLResponse)
def get_html():
    html_content = """
    <html>
        <head><title>My Custom Page</title></head>
        <body>
            <h1>Look, I'm returning a website!</h1>
        </body>
    </html>
    """
    return HTMLResponse(content=html_content, status_code=200)

@app.get("/plain", response_class=PlainTextResponse)
def get_plain_text():
    return "This is just boring, unformatted text."
In plain English: By declaring response_class=HTMLResponse, we guarantee FastAPI's Swagger docs correctly describe the route as returning HTML, rather than assuming it returns JSON.

RedirectResponse

Sometimes we want to bounce a user to a completely different URL. Maybe they visited an old, deprecated route, or maybe they just successfully logged in and we need to push them to the dashboard.

from fastapi.responses import RedirectResponse

@app.get("/old-route")
def redirect_to_new():
    # We use a 307 or 308 Temporary Redirect
    return RedirectResponse(url="/new-amazing-route", status_code=307)

FileResponse

If we want to let the user download a file stored directly on our hard drive (like a PDF report they generated), FileResponse is perfect.

It acts like a smart bouncer. It automatically reads the file size and sets the exact correct headers so the user's browser knows what kind of file it is downloading.

from fastapi.responses import FileResponse

@app.get("/download-report")
def download_file():
    # It automatically sets exactly the right Content-Type!
    return FileResponse(
        path="monthly_report.pdf", 
        filename="Your_Awesome_Report.pdf", # The name the user sees
        media_type="application/pdf"
    )

StreamingResponse

If we need to return a massive 50GB file, FileResponse might still struggle. Additionally, if we are returning a live video feed from a webcam, there is no physical file on the hard drive to send!

This is where StreamingResponse shines. We give it a Python generator (a function with yield), and it mathematically trickles the data to the browser piece by piece over a long period of time. 

from fastapi.responses import StreamingResponse

def fake_video_streamer():
    for i in range(10):
        yield b"chunk of video data\n"

@app.get("/video")
def get_video_stream():
    return StreamingResponse(
        fake_video_streamer(), 
        media_type="video/mp4"
    )

This ensures our server memory stays perfectly flat, no matter how gigantic the stream is.

Setting Custom Headers & Cookies Directly

Instead of manually constructing a full JSONResponse, FastAPI allows us to simply ask for the Response object directly inside our route. We can modify this object before returning our standard dictionary, and FastAPI smoothly merges them! 

from fastapi import FastAPI, Response

app = FastAPI()

@app.get("/set-cookie")
def set_cookie(response: Response):
    # 1. Modify the response before it goes out
    response.set_cookie(key="fakesession", value="fake-cookie-session-value")
    response.headers["X-Custom-Greeting"] = "Hello from FastAPI"
    
    # 2. Just return our normal dictionary!
    return {"message": "Look in your browser's dev tools!"}

This is incredibly useful when setting HTTP-only session cookies upon a successful login.

ORJSONResponse & ORJSONResponse

FastAPI uses the standard Python json library to encode outgoing responses. It is usually fast enough.

However, if our API needs to return a totally massive, 50,000-line dictionary, standard JSON serialization might slow us down. We can install hyper-optimized C++ backends instead!

pip install orjson or pip install ujson

from fastapi.responses import ORJSONResponse

@app.get("/heavy-data", response_class=ORJSONResponse)
def get_heavy_data():
    big_list = [{"id": i} for i in range(100000)]
    return big_list
In plain English: ORJSONResponse is significantly faster, but it doesn't always support the strangest, custom data types. Use it only on specific routes that are returning absolutely massive arrays of data.
