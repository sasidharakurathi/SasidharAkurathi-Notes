What is a Path Parameter?

Imagine reading a classic "fill-in-the-blank" sentence: "My favorite color is _______." The sentence structure never changes, but the meaning completely changes based on the word we put in the blank.

A Path Parameter is exactly the same concept, but for URLs. It allows us to create one single route that can handle thousands of different requests, just by filling in the blank.

In plain English: A path parameter is a dynamic placeholder inside our URL. It lets us capture specific information directly from the web address.

Basic Path Parameters and Types

To create a path parameter in FastAPI, we just wrap the dynamic part of the URL in curly braces {}. 

from fastapi import FastAPI

app = FastAPI()

@app.get("/users/{user_id}")
def get_user(user_id: int):
    return {"user": user_id}

Let's break down exactly what happens here:

Line 5: We define our route as /users/ followed by a blank space we name {user_id}.

Line 6: We tell the Python function to expect a variable named exactly user_id. Crucially, we add : int to tell FastAPI that this variable MUST be a whole number.

Line 7: We return the ID.

Why Do Types Matter So Much?

In older frameworks, everything pulled from a URL is passed to us as a simple text string (like "123"). If someone visited /users/apple, our code would try to process "apple" as a number and instantly crash our app.

FastAPI acts as a bouncer. By adding : int, FastAPI automatically checks the URL BEFORE running our code.

  • If they visit /users/3, it converts "3" into a real Python integer.
  • If they visit /users/apple, FastAPI automatically stops them and returns a clean, detailed error message saying: "Input should be a valid integer."
⚠️ Common Mistake: Forgetting to add Python type hints. If we just write def get_user(user_id): without the : int, FastAPI will just pass it as a raw string, and we won't get any automatic data validation!

Predefined Values (The Multiple Choice Test)

Sometimes, a fill-in-the-blank is too broad. We want to offer a multiple-choice question instead. Let's say we have an AI model API, and we only support three specific models. We don't want users typing in just anything.

We can enforce this using standard Python Enum classes.

from enum import Enum
from fastapi import FastAPI

class ModelName(str, Enum):
    alexnet = "alexnet"
    resnet = "resnet"
    lenet = "lenet"

app = FastAPI()

@app.get("/models/{model_name}")
def get_model(model_name: ModelName):
    if model_name is ModelName.alexnet:
        return {"message": "You picked AlexNet"}
    return {"model": model_name}

Here is what makes this so powerful:

Line 1: We import the built-in Enum tool from Python.

Line 4-7: We create a class that lists the only three acceptable options.

Line 11-12: We set up the path parameter, but notice how we type hint it differently! We write (model_name: ModelName). We are telling FastAPI: "This blank can only be filled by one of the options in our Enum class."

If the user asks for /models/resnet, it works. If they ask for /models/terminator, FastAPI automatically rejects it with an error saying that is not a valid choice. Even better, this dropdown menu automatically shows up in our Swagger UI (/docs) page!

Use this when: We have a strict, unchanging list of valid options that the user is allowed to request (like "small", "medium", "large").

File Path Parameters

There is a unique challenge when dealing with paths. What if the value we want to capture contains slashes inside it? For example, what if we want a user to ask for a file located at /files/documents/2026/report.pdf?

If we just use a standard path parameter, FastAPI will get confused by all the slashes. We have to tell it explicitly that this parameter is a "file path".

@app.get("/files/{file_path:path}")
def read_file(file_path: str):
    return {"file_path": file_path}

Line 1: Notice the special syntax :path inside the curly braces. This specific instruction tells FastAPI to ignore the forward slashes and capture the entire rest of the URL as a single string.

Advanced Numeric Validation (Greater Than, Less Than)

We already know how to make sure a path parameter is an integer. But what if we need to be stricter? What if the integer MUST be greater than zero, or less than 100?

To do this, we import a new tool directly from FastAPI called Path.

from fastapi import FastAPI, Path

app = FastAPI()

@app.get("/items/{item_id}")
def read_item(
    item_id: int = Path(title="The ID", ge=1, le=1000)
):
    return {"item_id": item_id}

This looks a little intimidating, so let's carefully break it down:

Line 1: We import the Path class directly from the main fastapi package.

Line 6: Notice that after the standard item_id: int, we add an equals sign, followed by Path(). This allows us to inject extra rules.

Line 7: Inside Path(), we pass our specific rules:

  • title: Adds a nice title in the Swagger UI documentation.
  • ge=1: This stands for Greater than or Equal to 1. The ID cannot be 0 or negative!
  • le=1000: This stands for Less than or Equal to 1000.

If a user visits /items/0, FastAPI instantly rejects them automatically before the Python function even runs.

In plain English: The Path() tool lets us add strict mathematical rules (like minimums and maximums) to our path parameters.

Here is a quick cheat sheet for the weird abbreviations:

  • gt: greater than (>)
  • ge: greater than or equal (>=)
  • lt: less than (<)
  • le: less than or equal (<=)
