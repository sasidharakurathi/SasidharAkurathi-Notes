The Missing Pieces

In the last chapter, we securely locked our doors using OAuth2PasswordBearer and get_current_user. But we faked the most important part: creating and verifying the actual JWT (JSON Web Token) "valet key".

To do real cryptography and password hashing safely, we should never write the math ourselves. We will rely on two industry-standard Python libraries. In our terminal, we need to run:

pip install "python-jose[cryptography]" "passlib[bcrypt]" python-multipart

  • passlib[bcrypt]: The ultimate tool for securely scrambling (hashing) user passwords so even if our database is stolen, hackers cannot read them.
  • python-jose: The tool that signs and verifies our glowing JWT stamps.

Hashing Passwords

When a new user registers, we take their plaintext password (like "ilovecats123") and pass it through bcrypt. It turns it into a chaotic string of characters. We save that chaotic string in our database.

When they log in later, we do not un-scramble the hash (that's impossible!). Instead, we take the new password they just typed, scramble it the exact same way, and see if the two chaotic strings match.

from passlib.context import CryptContext

pwd_context = CryptContext(schemes=["bcrypt"], deprecated="auto")

def verify_password(plain_password, hashed_password):
    return pwd_context.verify(plain_password, hashed_password)

def get_password_hash(password):
    return pwd_context.hash(password)
In plain English: We use passlib to turn passwords into unbreakable hashes, and to verify them without ever storing the real password.

Creating and Expiring the Token

A valet key is only safe if it automatically expires. We need to define a secret key (a massive, unguessable password that only our server knows) and an algorithm to digitally sign our JWTs so users cannot alter them.

import os
from datetime import datetime, timedelta
from jose import jwt

# In production, NEVER write your secret key in the code!
# Save it as an environment variable.
SECRET_KEY = "09d25e094faa6ca2556c818166b7a9563b93f7099f6f0f4caa6cf63b88e8d3e7"
ALGORITHM = "HS256"
ACCESS_TOKEN_EXPIRE_MINUTES = 30

def create_access_token(data: dict):
    to_encode = data.copy()
    
    # Calculate exactly when this token will die
    expire = datetime.utcnow() + timedelta(minutes=ACCESS_TOKEN_EXPIRE_MINUTES)
    to_encode.update({"exp": expire})
    
    # Cryptographically sign the token using our SECRET_KEY
    encoded_jwt = jwt.encode(to_encode, SECRET_KEY, algorithm=ALGORITHM)
    
    return encoded_jwt

Line 15: We calculate a strict death-date for the token (30 minutes from right now) and inject it into the payload under the standard "exp" key.

Line 19: We use python-jose to combine our data, the expiration time, and our massive SECRET_KEY to encode and securely sign the final JWT string.

⚠️ Common Mistake: Hardcoding the SECRET_KEY and pushing it to GitHub. If anyone discovers our secret key, they can forge JWTs and instantly have administrator access to our entire application.

The Complete End-to-End System

Let's combine everything we know: our token creator, our password verificator, OAuth2PasswordBearer, and our get_current_user dependency. This is a massive block of code, but it is the ultimate, production-ready foundation for our API's security.

from datetime import datetime, timedelta
from typing import Annotated

from fastapi import Depends, FastAPI, HTTPException, status
from fastapi.security import OAuth2PasswordBearer, OAuth2PasswordRequestForm
from jose import JWTError, jwt
from passlib.context import CryptContext

# 4. The Authenticator Dependency
async def get_current_user(token: Annotated[str, Depends(oauth2_scheme)]):
    credentials_exception = HTTPException(
        status_code=status.HTTP_401_UNAUTHORIZED,
        detail="Could not validate credentials",
        headers={"WWW-Authenticate": "Bearer"},
    )
    try:
        # A. Decode the JWT and verify the signature
        payload = jwt.decode(token, SECRET_KEY, algorithms=[ALGORITHM])
        username: str = payload.get("sub")
        if username is None:
            raise credentials_exception
    except JWTError:
        # B. If the token is expired or forged, ban them
        raise credentials_exception
        
    # C. Look the user up in the database
    user = fake_users_db.get(username)
    if user is None:
        raise credentials_exception
    return user

# 5. The Login Route (Generating the Token)
@app.post("/token")
async def login_for_access_token(
    form_data: Annotated[OAuth2PasswordRequestForm, Depends()]
):
    user_dict = fake_users_db.get(form_data.username)
    if not user_dict or not verify_password(form_data.password, user_dict["hashed_password"]):
        raise HTTPException(
            status_code=status.HTTP_401_UNAUTHORIZED,
            detail="Incorrect username or password",
        )
    # Give them the glowing stamp!
    access_token = create_access_token(data={"sub": user_dict["username"]})
    return {"access_token": access_token, "token_type": "bearer"}

# 6. The Secure Route
@app.get("/users/me")
async def read_users_me(
    current_user: Annotated[dict, Depends(get_current_user)]
):
    return current_user

Line-by-Line Breakdown

This is the most complex code we have run into, so let's break down exactly what happens when a user tries to log in and access their profile.

Step 1: The Login

  • The user sends their username and password to /token.
  • We use OAuth2PasswordRequestForm as a dependency. This magically creates a standard HTML form that the Swagger UI "Authorize" button knows how to use natively!
  • We look up the user. If they exist, we use verify_password() to compare the password they typed with the scrambled bcrypt hash in our database.
  • If it matches, we call create_access_token(). We put their username in the "sub" (subject) field, sign it with our secret key, and return the final JWT to the browser.

Step 2: The Secure Request

  • The user tries to visit /users/me.
  • The route is completely frozen by the Depends(get_current_user) lock.
  • FastAPI uses oauth2_scheme to extract the Bearer token from the headers and passes it to get_current_user.
  • Inside get_current_user, we use jwt.decode() to mathematically prove the token's signature was created by our SECRET_KEY. If the token was altered by a hacker, or if 30 minutes have passed and it is expired, jwt.decode() crashes with a JWTError.
  • We catch that exact error and throw a beautiful 401 Unauthorized exception, stopping the thief in their tracks.
  • If all is well, we extract the username from the "sub" field inside the token, fetch their profile from the database, and inject it seamlessly into our read_users_me function.
In plain English: We have built an incredibly strong, bank-grade vault door. Users check in at the front desk (/token), verify their password, and get a temporary, cryptographically signed badge (JWT). For the next 30 minutes, get_current_user checks that badge instantly for any secure room they try to enter.
