What is a Query Parameter?

If path parameters are like the delivery address on a package, query parameters are the extra delivery instructions. "Deliver to 123 Main St... but please leave it at the back door."

We see them every day. Whenever we search for something on Google or filter shoes by size on Amazon, the URL usually gains a question mark ? followed by key=value pairs. For example, /shoes?size=10&color=red.

In plain English: Query parameters are optional extra instructions added to the end of a URL after a question mark to filter, sort, or modify the response.

FastAPI is incredibly smart about this. If we declare a variable in our Python function, but it does NOT appear in our path decorator, FastAPI automatically assumes it is a query parameter.

from fastapi import FastAPI

app = FastAPI()

@app.get("/items/")
def read_items(skip: int = 0, limit: int = 10):
    return {"skip": skip, "limit": limit}

Let's look at how this works:

Line 5: We define a simple URL path /items/. There are no curly braces here!

Line 6: We add skip and limit to our function. Because they are not in the path (Line 5), FastAPI knows they must be query parameters. We give them default values of 0 and 10.

Line 7: We return the values back to the user.

Required vs Optional Parameters

When we provide a default value (like = 0), a query parameter becomes optional. If a user visits /items/, the API will happily use the defaults and return items 0 through 10.

However, if we do NOT provide a default value, the query parameter becomes strictly required. 

@app.get("/items/{item_id}")
def read_item(item_id: str, search: str):
    return {"item_id": item_id, "search": search}

In this example, search has no default value. If a user visits /items/foo-item, FastAPI will block the request and show a missing parameter error. The user MUST visit /items/foo-item?search=some-keyword.

⚠️ Common Mistake: Forgetting to add a default value to something meant to be optional. If we want an optional parameter but don't have a good default value to give it, use = None.

Boolean Magic

Sometimes we just want a simple "yes" or "no" toggle, like ?short=true. FastAPI handles Python bool (boolean) types beautifully. 

@app.get("/items/{item_id}")
def read_item(item_id: str, short: bool = False):
    if not short:
        return {"description": "This is an incredibly long description..."}
    return {"description": "Too long; didn't read."}

Line 2: We declared short as a boolean. Notice how clever FastAPI is: if a user visits the URL with ?short=1, or ?short=True, or ?short=true, or ?short=on, or ?short=yes... FastAPI automatically understands they all mean "True" and converts them perfectly.

Combining Path and Query Parameters

We can mix path and query parameters together in the same function. FastAPI is smart enough to sort them out automatically based on their names.

@app.get("/users/{user_id}/items/{item_id}")
def read_user_item(
    user_id: int, item_id: str, q: str = None, short: bool = False
):
    # 'user_id' and 'item_id' are path parameters
    # 'q' and 'short' are query parameters
    return {"user": user_id, "item": item_id, "q": q, "short": short}

Advanced Validation with Query()

Just like we used Path() in the last chapter to add extra validation to path parameters, we can use Query() to add extra validation to our query parameters. We import it from the main fastapi package.

String Validation

What if a search query is required, but it must be at least 3 characters long, and no more than 50 characters? 

from fastapi import FastAPI, Query

app = FastAPI()

@app.get("/items/")
def read_items(q: str = Query(default=None, min_length=3, max_length=50)):
    return {"q": q}

Line 6: Instead of writing q: str = None, we use Query() to set the default value, and then securely chain our length limits to it.

We can even enforce regular expressions using pattern="^fixedquery$" if we only want to accept very specifically formatted strings (like matching a specific employee ID format).

Numeric Validation

Just like with Path(), we can add greater-than and less-than rules to numbers using Query(). 

@app.get("/items/")
def read_items(price: float = Query(default=0, gt=0, lt=105.50)):
    return {"price": price}

Line 2: This ensures the price query parameter is strictly greater than 0, and strictly less than 105.50.

Lists of Multiple Values

Sometimes users want to filter by multiple things at once, like ?color=blue&color=red. To capture both, we just use a Python list!

from typing import List
from fastapi import FastAPI, Query

app = FastAPI()

@app.get("/items/")
def read_items(q: List[str] = Query(default=["foo", "bar"])):
    return {"q": q}

Line 7: By using List[str], FastAPI automatically gathers every ?q= parameter in the URL and hands them to us as a clean Python list. We also provided a default list of ["foo", "bar"] in case the user provides nothing at all.

Hiding and Deprecating

Finally, what if we are trying to phase out an old query parameter? We can pass deprecated=True into Query(). It will keep working, but it will show up as greyed-out and deprecated in our Swagger UI docs.

Or, what if we have a secret admin tool we do not want regular users to see in the documentation menu? We can pass include_in_schema=False to completely hide the query parameter from the automatic docs, while keeping it perfectly functional in the code!

In plain English: The Query() tool gives us superpowers. It allows us to strictly validate string lengths, limit number ranges, accept lists, and even hide parameters directly from the documentation.
