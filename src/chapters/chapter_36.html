The Language of the Internet

Whenever a client (a web browser, a mobile app, or another server) sends an HTTP Request to our API, we must reply with an HTTP Response. The single most important part of that response is the Status Code. This is a 3-digit number that instantly tells the client exactly how their request went.

In FastAPI, we should never hardcode raw numbers (like 200 or 404) into our code. Instead, we import the status module. It provides perfectly auto-completed, human-readable constants that make our code infinitely more professional.

from fastapi import FastAPI, status, HTTPException

app = FastAPI()

# Good: Clear, obvious, and auto-completed!
@app.post("/items/", status_code=status.HTTP_201_CREATED)
def create_item():
    pass

# Bad: What does 418 even mean?
@app.get("/teapot")
def make_tea():
    raise HTTPException(status_code=418, detail="I'm a teapot")

Status Code Categories

Status codes are universally grouped into 5 specific categories based on their starting number. As reliable senior developers, we absolutely must use the mathematically correct category every time.

  • 1xx (Informational): "Hold on, I'm still thinking." We almost never use these directly in building REST APIs.
  • 2xx (Success): "Everything went perfectly!" Our users love seeing these.
  • 3xx (Redirection): "Go look over there instead." Used to bounce traffic to new URLs.
  • 4xx (Client Error): "You messed up." The user forgot a password, typed a bad URL, or sent invalid JSON. The fault is entirely on them.
  • 5xx (Server Error): "We messed up." Our database crashed, our Python code threw a raw Exception, or our server caught on fire. It is our fault.

The Most Common 2xx Success Codes

FastAPI Constant When to Use It
status.HTTP_200_OK The absolute standard. Use this for almost every single successful GET, PUT, or DELETE request. It literally means "Yes, it worked."
status.HTTP_201_CREATED Use this exclusively for POST requests when a brand new resource (like a new User account) was successfully permanently saved in our database.
status.HTTP_202_ACCEPTED A rare gem. Use this when a user uploads a massive 4GB video, and our API says: "We received it and queued it up for background processing, but it's not actually done yet."
status.HTTP_204_NO_CONTENT Use this often for DELETE or PUT requests when the action succeeded, but we literally have zero JSON data to return in the body of the response.

The Most Common 3xx Redirection Codes

FastAPI Constant When to Use It
status.HTTP_301_MOVED_PERMANENTLY Use this when an old route is dead forever. If a browser hits it, Google's SEO bots update their massive web index to point to the new URL permanently.
status.HTTP_307_TEMPORARY_REDIRECT The absolute safest, standard way to temporarily bounce a user to a dashboard after logging in, without telling Google SEO bots to permanently rewrite their indexes.

The Most Common 4xx Client Error Codes

FastAPI Constant When to Use It
status.HTTP_400_BAD_REQUEST The standard "Catch-All" client error. Use this when the user's JSON was formatted correctly, but their business logic was awful (e.g., trying to buy a product that is out of stock).
status.HTTP_401_UNAUTHORIZED A terrible name for a vital code. This actually means Unauthenticated. Use this exclusively when the user provides the wrong password, or their JWT token is completely missing or expired.
status.HTTP_403_FORBIDDEN Use this when the user is perfectly authenticated (we know exactly who they are), but they are a basic user trying to access a restricted Admin-Only page. They lack authorization.
status.HTTP_404_NOT_FOUND Use this when they query /users/999 but User #999 does not exist in our Postgres database.
status.HTTP_409_CONFLICT Use this when a user attempts an action that violates database state, like trying to register a new account with an email address that is already globally taken.
status.HTTP_422_UNPROCESSABLE_ENTITY FastAPI returns this completely automatically! If a user sends a string "apple" into an integer field, Pydantic throws a 422 immediately. We rarely raise this manually.

The Most Common 5xx Server Error Codes

FastAPI Constant When to Use It
status.HTTP_500_INTERNAL_SERVER_ERROR The ultimate nightmare. FastAPI throws this automatically if our raw Python code crashes (e.g., trying to divide by zero). It fundamentally means our code is broken.
status.HTTP_503_SERVICE_UNAVAILABLE Use this to gracefully tell our users that our database is temporarily rebooting, or an external 3rd-party API we rely on is completely down. "We are okay, but our tools are broken."
