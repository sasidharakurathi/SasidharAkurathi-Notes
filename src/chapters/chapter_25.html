What Are Static Files?

So far, we have mostly returned JSON data from our API so that a separate frontend framework (like React or Vue) could consume it. But what if we want our FastAPI server to build and serve the literal, visual website directly to the user's browser?

Websites need images, CSS stylesheets, and plain JavaScript files. These files are "static"-meaning the server doesn't calculate or process anything inside them. It just grabs the file from the hard drive and sends it straight to the browser exactly as it is.

We absolutely could write an endpoint for every single image and CSS file using FileResponse, but that would take forever. Instead, we can tell FastAPI to "mount" a whole folder of files to our application.

In plain English: "Mounting" a folder means telling FastAPI: "Hey, if the user asks for any file starting with this URL, stop looking for API routes and just go find the file inside this specific hard drive folder."

Mounting StaticFiles

To serve static files, we use a specialized tool from Starlette called StaticFiles.

from fastapi import FastAPI
from fastapi.staticfiles import StaticFiles

app = FastAPI()

# Mount the folder!
app.mount("/static", StaticFiles(directory="static"), name="static")

Let's unpack Line 7:

  • "/static": The URL path. If a user's browser asks for http://ourstartup.com/static/logo.png, FastAPI takes over.
  • directory="static": The physical folder on our computer. We must create a folder named exactly "static" next to our main.py file, and put logo.png inside it.
  • name="static": An internal name for FastAPI (used mainly for reverse URL lookups).

Now, if we drop an image named welcome.jpg and a styling file named styles.css inside that folder, the browser can instantly download them at /static/welcome.jpg and /static/styles.css!

Serving a Frontend Automatically

If we want to build a Single Page Application (SPA), we could put our index.html (along with all its massive compiled React/Vue bundles) right into the static folder.

If we set html=True inside the StaticFiles declaration, FastAPI will automatically search that folder for an index.html file whenever the user visits that exact route.

# Serve our React application root immediately
app.mount("/", StaticFiles(directory="frontend/build", html=True), name="frontend")

This is brilliant for extremely small projects where we want our backend and compiled frontend served from the exact same python process!

Custom Paths for Uploads

Static files aren't just for CSS and JavaScript! Remember the file upload endpoints we built in Chapter 14?

If our API allows users to upload profile avatars, we could save those avatars into an uploads/ directory on our server. To let other users actually see those avatars on the frontend, we just mount the uploads folder!

import os
from fastapi import FastAPI
from fastapi.staticfiles import StaticFiles

# Make sure the directory actually exists
os.makedirs("user_uploads", exist_ok=True)

app = FastAPI()

app.mount("/media", StaticFiles(directory="user_uploads"), name="media")

Now, if User A uploads a picture and our code saves it as user_uploads/avatar_123.jpg, User B can view it in their browser simply by visiting /media/avatar_123.jpg.

Combining with HTML Templates

While React and Vue are massive, we don't always need them. If we are using FastAPI to render classic, server-side HTML (like we used to with Django or Flask using Jinja2 templates), static files are absolutely essential.

Inside our Jinja2 templates, we can use the exact name="static" from our mount command to dynamically generate the correct URL paths to our CSS and JS files.
